# Berry on Propeller 2 v0.9.4

This release hardens the Catalina P2 Edge workflow and makes the REPL much nicer for live board bring-up. It focuses on the no-PSRAM P2 Edge profile, pins 56/57 LED access, Hub-RAM size safety, runtime status reporting, command history editing, and cooperative Ctrl-C/Ctrl-D interrupts for runaway main-cog Berry code.

## Release Binaries

This release ships Catalina binaries:

- `berry_p2.binary`
  The normal Berry application image for RAM loading.
- `berry_p2_flash_loader.binary`
  The Catalina flash-programmer wrapper image generated from `berry_p2.binary`.

## Load to RAM or Flash

Load to RAM while developing:

```sh
make p2 TOOLCHAIN=catalina
make p2-run TOOLCHAIN=catalina PORT=/dev/cu.usbserial-P97cvdxp
```

Flash to the P2 Edge SPI flash:

```sh
make p2 TOOLCHAIN=catalina
make p2-flash TOOLCHAIN=catalina PORT=/dev/cu.usbserial-P97cvdxp
tio -b 230400 /dev/cu.usbserial-P97cvdxp
```

Important notes:

- Catalina is the preferred and verified way to compile Berry for P2.
- FlexC remains in the tree for legacy/debug work, but do not use it for normal Berry P2 builds.
- The default Catalina P2 Edge profile is the no-PSRAM board profile: `CATALINA_MODEL=COMPACT`, `CATALINA_CLIB=-lcx`, and no `-lpsram`.
- Pin 57 is usable as an LED on the no-PSRAM P2 Edge board; PSRAM builds reserve it as the PSRAM chip-select.
- P2 app images are checked against the 512 KiB Hub RAM limit before `berry_p2.binary` is published.

## New Features

### No-PSRAM P2 Edge GPIO Profile

The Catalina defaults now target the no-PSRAM P2 Edge LED layout. Pins 56 and 57 are available for normal GPIO when PSRAM support is not enabled.

```berry
import p2

p2.pinmode(57, p2.OUTPUT)
p2.high(57)
p2.sleep_ms(250)
p2.low(57)
```

### `p2.status()`

`p2.status()` prints a small board/runtime report with image size, heap bars, clock information, and all eight cog states.

```berry
import p2

p2.status()
```

It reports lines like:

```text
Memory
  image        [#######################.]  95%  498400 /  524288 B
  main heap    [........................]   1%    1184 /  131072 B
  worker heap  [........................]   0%       0 /   32768 B
```

### Mini Readline

The P2 REPL now supports a tiny line editor. Up/Down cycle through the last three commands, Left/Right move within the line, and Backspace/Delete edit recalled commands before Enter.

```berry
p2.low(56)
```

Press Up, change `56` to `57`, then press Enter to run the edited command.

### Ctrl-C/Ctrl-D Main-Cog Interrupts

Ctrl-C or Ctrl-D now interrupt runaway Berry code running on the main REPL VM and return to the prompt with a short `KeyboardInterrupt` message.

```berry
import p2

while true
    p2.toggle(56)
    p2.sleep_ms(100)
end
```

Worker cog loops stay separate. The interrupt hook records the main cog at startup and ignores worker-cog VMs; use the worker API to stop worker jobs.

### Hub-RAM Size Guard

The P2 build refuses to publish an oversized app image. If a build exceeds the 512 KiB Hub RAM limit, the build fails and removes the oversized output instead of leaving a stale or unusable binary behind.

```sh
make p2 TOOLCHAIN=catalina
```

## Existing Features

### `p2` Hardware Module

The `p2` module remains the friendly hardware namespace for GPIO, delays, cog helpers, locks, heap reporting, status, and tone output.

```berry
import p2

p2.pinmode(56, p2.OUTPUT)
p2.toggle(56)
print(p2.cogid())
```

### Worker Cog

`worker.start()` launches a second cog with its own Berry VM and worker heap. The v1 worker keeps name-based integer dispatch.

```berry
import worker

print(worker.start())
worker.exec("blink", 56, 250)
print(worker.state())
worker.stop()
```

### I2C, SPI, Threads, And Spin2

The `i2c`, `spi`, `threads`, and `spin2` modules from v0.9.3 remain available for board bring-up, fixed-channel messaging, and compatible Spin2 mailbox binaries.

```berry
import i2c

i2c.init(25, 24, 400)
print(i2c.scan())
```

## Verification Notes

- Built with Catalina 8.8.9 through Docker.
- RAM-loaded on P2 Edge using `/dev/cu.usbserial-P97cvdxp`.
- Verified the release image fits Hub RAM: `498400 / 524288` bytes.
- Verified the REPL starts correctly on the no-PSRAM `P2_EDGE` profile.
- Verified Ctrl-C interrupts both `while true; end` and `while true; p2.sleep_ms(100); end` on the main REPL VM.
- Verified Ctrl-C returns to `berry>` with `KeyboardInterrupt` instead of a traceback.
- Verified the worker cog path remains explicitly controlled through the worker API.

## V1 Limitations

- Ctrl-C/Ctrl-D interruption is cooperative. Native calls that do not poll can still block until they return.
- Main-cog interrupts do not stop worker cog loops by design.
- Worker and `p2.cog_start()` dispatch only by function name and integer arguments.
- `threads` is a small channel/key-value layer, not a full scheduler yet.
- `spin2.call()` requires binaries that implement the documented Berry mailbox convention.
