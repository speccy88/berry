# Release Notes Policy

Propeller 2 releases in this fork should be written as mini tutorials, not just change logs.

From this point forward, each release note should include:

- a `New Features` section
- an `Existing Features` section
- a short explanation for every listed feature
- a small working example for every listed feature
- a `Load to RAM or Flash` section that explains which binary is used for each flow

The goal is that someone can open a release page and immediately understand what this Berry-on-P2 port can do today, even if they did not read the previous release.
