# Berry on Propeller 2 v0.9.5

This release focuses on the Spin2 module test suite and the SD-card deployment
workflow for Propeller 2.

## Spin2 Build Layout

`make spin2` now writes SD-safe uppercase DOS 8.3 binary names into
`spin2/build/*.BIN`. The generated `spin2/build/MANIFEST.TXT` maps each source
file to its binary name, and FlexSpin `.p2asm` listings are moved out of the
binary folder into `spin2/build/p2asm`.

This avoids Catalina DOSFS aliases such as `MB_1~109.BIN` when the files are
copied to or loaded from the SD card.

## Hardware-Tested Mailbox Binaries

The PASM2 mailbox suite was transferred to a P2 Edge SD card and run from the
flashed Catalina Berry image. All mailbox binaries passed `spin2.start()`,
`spin2.call()`, and `spin2.stop()` testing, including:

- `MBOXDEMO.BIN`
- `MB_01ALU.BIN` through `MB_20PAT.BIN`
- `P2INSTMX.BIN` start/stop smoke coverage

The detailed per-binary table is in `spin2/SPIN2_BINARY_TEST_REPORT.md`.

## Fixes

- Fixed the shared Spin2 mailbox service so method return flags no longer
  corrupt the method-dispatch decision. This fixes the previous
  `MB_04CMP.BIN` runtime failure.
- Updated the SD loader host to transfer uppercase `.BIN` files.
- Added conservative serial pacing for SD transfers.
- Switched the temporary RAM SD loader to unmanaged Catalina file I/O.

## Known Limits

The `s2_*.spin2` files compile and document Spin2 language features. Their
high-level FlexSpin binaries contain absolute Hub-address references, so Berry
detects and rejects them as non-relocatable instead of launching them. Runtime
`spin2.call()` coverage remains limited to the `mb_*.BIN` mailbox suite.

The temporary SD loader also hit Catalina DOSFS create limits after about 13
files in one fresh subdirectory on the tested card, so the hardware run used
multiple batch directories. The normal single-file path remains useful, and the
batch behavior is documented in the Spin2 binary test report.
